## 導入文テンプレート（PASONA法）

═══════════════════════════════

正本：writing-manual.md §A-2

### 構成

```
[1文目] P（Problem）: メインKWを含む悩み提起
[2〜3文目] A（Agitation）: 放置リスクを示唆
[4文目] S（Solution）: 「この記事では〜を解説します」で統一
[toc]
```

### 具体例：プログラミングスクール 東京

```
プログラミングスクールを東京で探していても、どこを選べばいいか迷っていませんか？

スクールの料金は数万円から数十万円まで幅があり、自分に合わないものを選ぶと費用も時間も無駄になります。

受講後に後悔しても、払った受講料は戻ってきません。

この記事では、東京で通えるプログラミングスクール30選を紹介し、選び方のポイントを解説します。

[toc]
```

### 具体例：生成AIスクール

```
生成AIを学べるスクールが増えていますが、どこが自分に合うか判断しづらくありませんか？

カリキュラム内容や学べるAIツールはスクールごとに大きく異なります。

目的に合わないスクールを選ぶと、数十万円の受講料が無駄になってしまいます。

この記事では、目的別におすすめの生成AIスクール10選を紹介します。

[toc]
```

### ⚠️ 導入文ルール

- テックジムへの言及は入れない（本文中でテックジム除外方針）
- 「本記事では」ではなく「この記事では」で統一
- 「ぜひ参考にしてください」「参考になれば幸いです」等の定型文は使わない

## 選び方 H3 のテンプレート（PREP法）

═══════════════════════════════

正本：writing-manual.md §E-2補足

### 構成

```
[1文目] P: 結論（このH3で言いたいことの要点。メインKW/複合KW含む）
[2文目] R: 根拠・理由（なぜそれが重要か）
[3〜4文目] E: 具体例（数値・固有名詞・条件のいずれかを含む）
[最終文] <strong>P: strong締め（「何を・どこで・どうやって」の2要素以上）</strong>
```

### ボリューム

- 4〜6文以内（strong 締め含む）
- 一文一改行（「。」の直後に空行）
- 1文60字以内

### 具体例：「受講形式が自分のライフスタイルに合っているか」

```
オンライン完結型か通学型か、受講形式を事前に確認してみてください。

仕事をしながら学ぶ方は、平日夜や週末に受講できるスケジュールかどうかが重要です。

通学型のスクールは対面で質問しやすいメリットがある一方、通学時間がかかります。

オンライン型は場所を選ばず学べますが、自己管理が求められます。

<strong>公式サイトの受講スケジュール欄で、自分の生活リズムに合うか確認してみてください。</strong>
```

### 具体例：「給付金制度に対応しているか」

```
教育訓練給付金（国が受講料の一部を補助する制度）の対象講座なら、受講料の最大70%が国から給付されます。

スクールによって対象コースと非対象コースが分かれているため、事前確認が必要です。

給付金の申請にはハローワークでの手続きが必要で、受講開始の1ヶ月前までに申し込みます。

<strong>給付金の対象講座かどうかは、厚生労働省の [講座検索システム](https://...) で確認できます。</strong>
```

⚠️ 「教育訓練給付金」のような制度名は初出時にカッコで補足する（writing-manual §E-8）。

## メリット H3 のテンプレート（PREP法）

═══════════════════════════════

### 構成

```
[1文目] P: 結論（このメリットの要点。メインKW/複合KW含む）
[2文目] R: 根拠・理由
[3〜4文目] E: 具体的な効果（読者の生活・キャリアでの変化）
[最終文] <strong>P: strong締め（読者にとっての価値）</strong>
```

### 具体例：「仕事と並行して学習を進めやすい」

```
プログラミングスクールの多くは、仕事と並行して学べるカリキュラムを用意しています。

オンライン教材なら通勤時間や昼休みにも学習を進められます。

自習型のスクールでは決まった時間割がないため、急な残業があっても振替の必要がありません。

<strong>仕事を辞めずにスキルを習得できるため、収入を維持したまま転職準備ができます。</strong>
```

## 注意点 H3 のテンプレート（PREP法）

═══════════════════════════════

### 構成

```
[1文目] P: 結論（この注意点の要点。メインKW/複合KW含む）
[2文目] R: 失敗パターン（読者のあるあるシーン）
[3〜4文目] E: 対策（具体的な行動）
[最終文] <strong>P: strong締め（対策の効果 or 具体的な行動）</strong>
```

### 具体例：「受講料以外のコストも含めて比較する」

```
プログラミングスクールの料金を比較するときは、受講料だけでなく入会金・教材費・PCレンタル代も含めた総額で比べてみてください。

受講料が安く見えても、入会金が数万円かかるスクールもあります。

公式サイトの料金ページで「税込総額」と「追加で必要な費用」の両方を確認してください。

<strong>総額で比較すれば、本当にコストを抑えられるスクールが見えてきます。</strong>
```

## サービス紹介 H3 のテンプレート（FAB法）

═══════════════════════════════

正本：writing-manual.md §B-2

### 構成

```
[段落1] F（Feature）: <strong>サービスの特徴を1つ（複合KW含む）</strong>
  - 特徴は1つに絞る。複数の情報を入れる場合は2文に分割する

[段落2] A（Advantage）: 他サービスとの違い・独自性
  - 具体的に何が違うか
  - 比較対象を明示する

[段落3] B（Benefit）: 読者にとっての価値
  - 機能の羅列ではなく、読者の生活・キャリアでの効果に変換

[最終行] CTA（cta-templates.md からコピペ）
```

### ⚠️ 2026-04 の変更（重要）

- **段落4（キャンペーン・給付金情報）は廃止**。必要な場合は段落2〜3に組み込む。
- **テックジムは本文中で出さない**。サービス紹介には含めず、フッターに集約。

### 具体例（テーブルなし）

```
<a href="https://af.moshimo.com/af/c/click?a_id=5222640&amp;p_id=2717&amp;pc_id=6139&amp;pl_id=39604" rel="nofollow">RUNTEQ</a>は、<strong>Webエンジニアへの転職に特化したプログラミングスクール</strong>です。

一般的な学習型スクールと違い、実務に近い開発環境でチーム開発を体験できる点が特徴です。

受講中にポートフォリオ（作品集）を複数作成でき、転職活動で実績としてアピールできます。

<a href="https://af.moshimo.com/af/c/click?a_id=5222640&amp;p_id=2717&amp;pc_id=6139&amp;pl_id=39604" rel="nofollow">RUNTEQ 公式サイト</a>
```

### 具体例（テーブルあり・大規模記事）

```
<!-- wp:heading -->
<h2>■RUNTEQ｜Webエンジニア転職特化のプログラミングスクール</h2>
<!-- /wp:heading -->

[画像：service-info.md の画像]

RUNTEQは、<strong>Webエンジニアへの転職に特化したプログラミングスクール</strong>です。

一般的な学習型スクールと違い、現役エンジニアによるレビューを受けながらチーム開発を経験できます。

受講中に複数のポートフォリオ（作品集）を作成でき、転職活動で「実務経験レベル」としてアピールできます。

[3行スペック表：場所・料金・学べる言語]

RUNTEQの詳細が知りたい方は下記をご覧ください。

[VKボタンCTA：cta-templates.md からコピペ]
```

### 書き出しパターン（最低3種類を使い分ける）

```
パターン①：「サービス名は、〜特徴〜のプログラミングスクールです。」
パターン②：「〜特徴〜を提供するサービス名は、」
パターン③：「サービス名の最大の特徴は、〜です。」
パターン④：「月額〇〇円から受講できるサービス名は、」（安さ特化記事）
```

### ⚠️ チェックポイント

- [ ] 段落1のstrong 1文目に複合KW（「プログラミングスクール」「生成AIスクール」等）が含まれている
- [ ] 段落1の特徴が1つに絞られている
- [ ] 段落2で他社との違いを具体的に述べている
- [ ] 段落3で読者メリットに変換されている（機能の羅列で終わっていない）
- [ ] テーブルに記載した情報（料金・言語等）を本文で同じ角度で繰り返していない
- [ ] 書き出しパターンが同じ記事内で3種類以上ある
- [ ] **テックジムへの言及が入っていない**
- [ ] CTAがサービス紹介文の最終行に配置されている
- [ ] 1サービスにつきCTAが1つのみ

## FAQ H3 のテンプレート

═══════════════════════════════

### 構成

```
[H3] Q. 質問文（「〜ですか？」「〜ますか？」で統一）
[1文目] 回答の結論
[2〜3文目] 補足・具体的な情報
```

### ルール

- 本文で既に扱った内容の焼き直しは禁止
- サジェスト・知恵袋・PAA・競合のFAQから、本文で未対応のニーズを選ぶ
- 質問の語尾は「〜ですか？」「〜ますか？」で統一（丁寧語）
- 知恵袋への言及は入れない

### 具体例

```
<h3>Q. 未経験からでもプログラミングスクールに通えますか？</h3>

はい、多くのスクールが未経験者向けのカリキュラムを用意しています。

RUNTEQでは受講生の9割以上がプログラミング未経験からスタートしています（[RUNTEQ 公式](https://...)）。

無料体験やカウンセリングを受ければ、自分のレベルに合うか事前に確認できます。
```

## まとめ H2 のテンプレート

═══════════════════════════════

正本：writing-manual.md §A-5

### 構成

```
[1文目] <strong>記事全体の結論を1文で要約</strong>
[2文目] <strong>読者にとっての最も重要なポイント</strong>
[3文目] 具体的な次のアクション（無料カウンセリング・体験レッスン等）
```

### ボリューム

- 3〜5文で短く締める
- 本文の繰り返しにしない

### 具体例

```
<strong>プログラミングスクールは料金・受講形式・サポート体制が大きく異なるため、自分の目的に合ったスクールを選ぶのが重要です。</strong>

<strong>給付金制度を活用すれば自己負担を大幅に抑えてスキルを習得できます。</strong>

まずは気になるスクールの無料カウンセリングに参加して、カリキュラムや雰囲気を確認してみましょう。
```

### ⚠️ 2026-04 の変更

- **テックジムへの言及は入れない**（本文中テックジム除外方針。フッターCTAに任せる）
- 「いかがでしたか」「参考になれば幸いです」等のAI定型で終わらない
