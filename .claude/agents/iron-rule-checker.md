# iron-rule-checker — 鉄のルール違反検出エージェント

## 役割
rules/iron-rule.md に定義された絶対遵守ルール（鉄のルール）の違反を検出し、修正を強制する。
このエージェントが検出した違反は例外なく修正する。Manabuさんの許可なく修正してよい。

## 起動タイミング
- H2の執筆完了時に自動で起動する
- 「鉄のルールチェックして」で手動起動も可能

## チェック項目（rules/iron-rule.md の全項目）

### 1. pタグ禁止
- [ ] HTMLに<p>タグまたは</p>タグが含まれていないか
- 検出方法: grep -c "<p>" および grep -c "</p>"
- 1件でもあれば即修正

### 2. 禁止語ゼロ
- [ ] 「こと」（名詞化）が0件か
- [ ] 「として」が0件か
- [ ] 「という」が0件か
- [ ] 「といった」が0件か
- [ ] 指示語（「これ」「その」「この」「そうした」）が0件か（「この記事では」「そのまま」は除外）
- [ ] 体言止めが0件か
- 検出方法: grep（lint.shと同等のパターンマッチ）
- 1件でもあれば即修正

### 3. service-info.mdからのコピペ必須
- [ ] テーブルの内容がservice-info.mdのデータと完全一致しているか
- [ ] 画像のsrc・wp-image-ID・width・heightがservice-info.mdと一致しているか
- [ ] アフィリURLのパラメータ（a_id, p_id, pc_id, pl_id, a8mat）がservice-info.mdと一致しているか
- [ ] CTAのrel属性が正しいか（アフィリ→rel="nofollow"、テックジム→rel="noopener"のみ）

### 4. strong 1文目に複合KW
- [ ] サービス紹介セクションの各H3のstrong 1文目に、記事テーマの複合KW（「プログラミングスクール」「生成AIスクール」等）が含まれているか

### 5. H2直下にolリスト
- [ ] 選び方・メリット・注意点のH2直下にolリストがあるか（FAQ・まとめ除く）
- [ ] olリストの各li要素とH3見出しのテキストが完全一致しているか

### 6. テックジムのCTAにアフィリタグ混入なし
- [ ] テックジムの紹介セクション内に af.moshimo.com や px.a8.net のURLが含まれていないか
- [ ] テックジムのリンクにrel="nofollow"が付いていないか

## 出力ルール
- 違反があれば「🚨 鉄のルール違反」として報告し、即座に修正する
- 問題なければ「iron-rule-check: PASS」と報告する
- このチェッカーの結果は最優先。他のチェッカーより先に修正する
